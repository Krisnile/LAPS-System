LAPS — User documentation

LAPS is a lightweight, prompt-based image segmentation labeling system (e.g. SAM); model backends may be extended (e.g. YOLO) for assisted annotation. This page documents module roles, dependencies, and navigation.

Structure and dependencies

Datasets can be created first; projects then declare task type and optional dataset links; tasks and annotation consume those entities.

  1. Dataset — upload and manage images freely; no project is required on the Datasets page.
  2. Project — top-level container: pick an annotation task type (e.g. SAM segmentation) and optionally link one or more datasets.
  3. Task — binds one project, one image, and workflow state (e.g. new, done). Annotation operates on this binding.
  4. Annotate — interactive segmentation workspace (SAM). Opened from the Dashboard or Task list via the Annotate action for a selected task.

Dashboard (home) provides aggregate metrics, quick links to Projects, Datasets, and Tasks, and a recent-task list with direct entry to Annotate per task.

Standard workflow

  1. Datasets — list at /datasets/ (basic info + new import); open each dataset’s detail URL (e.g. /datasets/123/) to append files, preview images, edit captions, and edit metadata. Link projects on Projects or Tasks. /datasets/
  2. Projects — create or edit a project (name, description, annotation task type, linked datasets, optional label configuration). Deleting a project removes its tasks. /projects/
  3. Tasks — see task-type samples and quick-create a project; batch-generate tasks by choosing a project and dataset (if the project has linked datasets, only those appear). /tasks/
  4. From the Dashboard or Task list, open Annotate for a task. The URL may include task_id to bind the image context.
  5. Annotate — Run / Save / Undo / Clear per toolbar; Prev / Next when a task queue is available. /annotate/ — 详见下文「标注页」

Annotate page — workflow and user guide

This section replaces the former on-page help modal: all rules, layout notes, and troubleshooting for the annotation workspace are kept here.

Data model and business rules

  • You must have at least one project. Projects may link zero or more datasets; if linked, batch task creation elsewhere only offers those datasets (same rule as the Tasks page).
  • After linking, you can create tasks for images (one task per image per project). The same image cannot have two tasks under the same project.
  • If the project has no linked datasets, the system may still let you pick any of your datasets when creating tasks, consistent with the Tasks page.
  • On large screens the canvas is on the left and the project/task panel on the right; on small screens the panel stacks above the canvas so you can pick a task first.

Recommended steps on /annotate/

  1. Expand the workflow strip at the top if needed; use the quick links (Datasets / Projects / Tasks) to fix missing data.
  2. In the right panel tab "Project & tasks", choose a project, then click a task row to load its image into the canvas.
  3. Switch to the "Labels (save)" tab if you need to pick foreground / background / other before saving the mask.
  4. Use point or box mode, add prompts on the image, press Run (SAM), then Save to store the mask for the current task.

Toolbar, zoom, and keyboard shortcuts

  • Point / Box — toggle prompt type; Run — call segmentation; Save — upload mask; Undo — step back; Clear — remove prompts (not the saved mask file until you overwrite).
  • Keyboard: Space = Run, S = Save, Ctrl+Z (or Cmd+Z) = Undo, C = Clear prompts. Buttons show hints in their title attributes.
  • Zoom controls and mask opacity slider affect display only (opacity of the overlay preview).
  • The prompts list under the canvas summarizes current points/boxes.

Labels before save

Select a label row (foreground / background / other) before saving so the server can store the intended class with the mask. If unsure, keep the default and still save.

New task modal

Opens from the right panel when a project is selected. Each tile is an image that does not yet have a task in this project; picking one creates a single task. The same image cannot be added twice to the same project.

Troubleshooting

  • No projects — 去项目页创建。
  • Project has no linked datasets — 编辑项目并勾选数据集；if none are linked, you may still use any of your datasets when creating tasks (same as Tasks page).
  • No images — 数据集页上传。
  • Task list empty — 使用右侧「新建任务」，或在 任务页批量生成。
  • Cannot save — Ensure a task is selected on the right, the image is loaded, and you have run segmentation at least once; check that you are logged in.

Alignment with Label Studio–style flow

Label Studio centers on a Project, Data Manager, and per-task labeling. Here, Projects carry an annotation type and optional many-to-many linked datasets; Datasets are managed freely; batch task creation restricts dataset choice when the project has linked datasets.

Sidebar — module reference

  • Dashboard — summary metrics, shortcuts, recent tasks with per-task Annotate links.
  • Projects — project CRUD with annotation type and linked datasets.
  • Datasets — full lifecycle for datasets and images without mandatory project binding.
  • Tasks — task CRUD (project, image, status, assignee). Annotate resolves the active task from the selected row.
  • Annotate — opens the workspace. If no valid task context is loaded, return to Tasks or the Dashboard, select a task, then use Annotate.

Top bar — account and preferences

Independent of the labeling workflow: the account menu provides profile settings and Logout. Theme accent, light/dark mode, and UI language are configured on the top bar.
