LAPS — User documentation

LAPS is a lightweight, prompt-based image segmentation labeling system (e.g. SAM); model backends may be extended (e.g. YOLO) for assisted annotation. This page documents module roles, dependencies, and navigation.

Structure and dependencies

Recommended data flow: create datasets, then projects (with optional dataset links), then tasks, then label on /annotate/. The list below is the single place where cross-entity rules are spelled out.

  1. Dataset — stores images; managed on the Datasets page without requiring a project.
  2. Project — annotation task type plus optional links to one or more datasets. If linked, batch task creation (Tasks page and similar) only lists those datasets; if not linked, you may still choose any of your datasets when creating tasks.
  3. Task — binds one project and one image; the same image cannot appear in two tasks under the same project.
  4. Annotate — workspace at /annotate/ for choosing project and task, running segmentation, saving, and exporting. Entry points: sidebar, Dashboard recent tasks, or Tasks list. 界面与操作见下文。

The Dashboard shows recent tasks with links into Annotate; it does not replace the project/task picker on /annotate/.

Standard workflow

  1. /datasets/ — Create datasets, upload images, edit captions and metadata on each dataset’s detail page.
  2. /projects/ — Create or edit projects (annotation type, optional dataset links, label defaults). Deleting a project removes its tasks.
  3. /tasks/ — Batch-generate tasks by project and dataset, respecting linked-dataset rules above. Alternatively, on /annotate/ use New task after selecting a project.
  4. /annotate/ — Label; details in 「标注页 — 工作流与操作说明」。

Annotate page — workflow and user guide

This section covers /annotate/ only: layout, controls, persistence, export, and troubleshooting. Dataset / project / task relationships are defined under Structure and dependencies; they are not repeated here.

页面布局（自上而下）

  1. Annotation workflow card — collapsible checklist with links to Datasets, Projects, and Tasks.
  2. Annotation toolbar card — header (title and doc links) and a body that lists: category name (for the next Run), segment roles, segmentation model, export format, Export, and mask opacity (display only).
  3. Main row — wide screens: canvas card on the left, project/task sidebar on the right; narrow screens: sidebar stacks above the canvas.
  4. Canvas card — image and overlay; under the image: Point/Box/Brush/Eraser, Run, Save, Undo, Clear all prompts, zoom and reset, mask opacity, brush size (visible in Brush/Eraser mode); below that: Completed runs, Current prompts, and current task summary.

仅本页生效的约定

  • Select a project in the sidebar before the task list and New task unlock.
  • A yellow notice may appear when the project has no linked datasets; task creation options still follow the global rules in Structure and dependencies.
  • Model choice: SAM returns a segmentation mask via prompt-driven interaction. YOLO11-seg performs automatic instance segmentation on the full image; you can optionally add foreground/background points or a box to filter which detected instances to keep. Both models return a grayscale mask PNG. Weights go under model/sam/ and model/yolo/ respectively.sk PNG; YOLO runs derive it from the union of detected boxes.

典型标注流程

  1. Sidebar: select project, then a task to load the image.
  2. Toolbar card: set segment role and optional category for the next Run; choose the segmentation model. Semantics: 「角色与类别名」。
  3. Canvas area: choose Point or Box, place prompts, Run. Both SAM and YOLO11-seg runs add a colored mask overlay—each adds one row in Completed runs. Repeat as needed with new role, category, model, or prompts. After running, switch to Brush or Eraser mode (B / E key) to manually refine the mask — paint to add regions, erase to remove. Scroll wheel zooms the canvas; Shift+scroll or [ ] adjusts brush size.
  4. Save writes only runs that are not yet in the database (see next heading). Export packages every saved annotation in the current project; format details are under Export formats.

保存、重载、列表行与删除

  • Save creates one database annotation per unsaved run on the current task and sets the task to Done. If all runs are already saved, the UI reports nothing new to upload.
  • Switching tasks and returning reloads masks and rows from the server for that task.
  • Delete on an unsaved row removes only that overlay locally. Delete on a saved row removes the annotation in the database; if it was the last annotation on the task, the task may return to Pending.

快捷键与仅影响显示的控件

  • Space = Run, S = Save, Ctrl+Z or Cmd+Z = Undo (last prompt step), C = Clear all prompts, P = Point mode, X = Box mode, B = Brush mode, E = Eraser mode, [ = shrink brush, ] = grow brush. Button titles carry the same hints. Scroll wheel zooms; Shift+scroll adjusts brush size.
  • Clear all prompts clears pending points/boxes before the next Run only; it does not remove saved overlays.
  • Zoom and mask opacity adjust on-screen preview only, not stored mask files.

画布下方的列表

  • Completed runs — one line per Run (or per saved annotation after reload); category, role, and prompt summary; row delete behavior is described under Save, reload…
  • Current prompts — points/boxes waiting for the next Run only.

角色与类别名（导出中的含义）

Segment role (foreground / background / other) is chosen before each Run and stored as metadata on the saved annotation. Category name applies to the next Run; if empty, the stored label is default. Roles are not three separate COCO class names—they annotate intent and may appear as sidecar fields (e.g. segment_role) in merged exports.

For multiple instances on one image with distinct class names in downstream tools, use a different category name before each Run; roles can still record UI intent separately.

导出格式（项目范围）

Requires a selected project. The download includes every annotation already saved in that project; unsaved overlays on the canvas are not included.

  • COCO JSON — one merged file with multiple images and instances; categories reflect stored names (and roles where the format carries them).
  • Simple JSON (LAPS) — one file listing per-annotation records (paths, RLE, etc.).
  • Pascal VOC — ZIP; one XML per source image, multiple object nodes when an image has several instances.
  • YOLO bbox — ZIP; one txt per image, one line per instance (boxes derived from masks).
  • Mask PNG — ZIP of raster files named by task and annotation id.

The category field starts empty on each visit to the annotate page; it is not saved in the browser when you leave or refresh.

「新建任务」弹窗

Shown when a project is selected. Tiles are images that have no task in this project yet; one click creates one task. Images already tied to a task in this project never appear in the grid.

常见问题与排查

  • Missing prerequisites — Create projects, datasets, or tasks via 项目、 数据集、 任务； or use New task on Annotate after picking a project.
  • Linked-dataset notice — Optional: attach datasets on the project edit form. Behavior without links is defined under Structure and dependencies, not duplicated here.
  • Save reports nothing to save — You need a selected task, at least one new Run after the last save, and a segment role chosen before each Run. See Save, reload…
  • Export unavailable or empty — Choose a project first. If nothing has been saved yet, archives may contain only a readme or minimal JSON.

与 Label Studio 类产品对照

Those products usually pair a project with a data manager and per-task UIs. LAPS separates free-form dataset storage from projects that declare an annotation type and optional dataset links; the exact linking and batch rules are only in Structure and dependencies above.
